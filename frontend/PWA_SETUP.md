# PWA Setup for Laamly

This app has been configured as a Progressive Web App (PWA) according to Mozilla documentation.

## What's Included

✅ **Web App Manifest** (`/public/manifest.json`)
- App name, description, and branding
- 192x192 and 512x512 icons (required for installability)
- Standalone display mode
- Theme colors and orientation settings

✅ **Service Worker** (auto-generated by vite-plugin-pwa)
- Automatic updates
- Offline caching
- Runtime caching for API calls

✅ **PWA Icons**
- `/public/icon-192x192.png`
- `/public/icon-512x512.png`
- Apple touch icon support

✅ **HTML Meta Tags**
- Manifest link
- Theme color
- Meta description
- Apple touch icon

## Testing the PWA

### Development
```bash
npm run dev
```
Visit https://localhost:5177 (PWAs require HTTPS or localhost)

### Production Build
```bash
npm run build
npm run preview
```

### Installation Prompt
On supported browsers (Chrome, Edge, Safari on macOS 17+), users will see an install prompt when they visit the app. The app can be installed to:
- Desktop (Windows, macOS, Linux)
- Mobile (Android, iOS 16.4+)

### Testing PWA Features

1. **Chrome DevTools**
   - Open DevTools (F12)
   - Go to Application tab
   - Check "Manifest" section - should show all manifest details
   - Check "Service Workers" section - should show registered service worker
   - Use Lighthouse tab to audit PWA score

2. **Install the App**
   - In Chrome/Edge: Look for install icon in address bar
   - In Safari (macOS): File > Add to Dock
   - On Mobile: Use browser's "Add to Home Screen" option

3. **Offline Testing**
   - Install the app
   - Open DevTools > Network tab
   - Check "Offline" mode
   - App should still load cached assets

## PWA Requirements Met

According to [Mozilla's PWA guide](https://developer.mozilla.org/en-US/docs/Web/Progressive_web_apps/Guides/Making_PWAs_installable):

- ✅ Web app manifest with required fields (name, icons, start_url, display)
- ✅ 192px and 512px icons
- ✅ HTTPS/localhost requirement (enforced in production)
- ✅ Service worker for offline support
- ✅ `prefer_related_applications` set to false

## Customization

To customize the PWA:

1. **Replace Icons**: Replace `/public/icon-192x192.png` and `/public/icon-512x512.png` with your branded icons
2. **Update Colors**: Edit theme colors in `/public/manifest.json` and `index.html`
3. **Cache Strategy**: Modify workbox configuration in `vite.config.ts`
4. **App Name**: Update name in manifest and HTML title

## Browser Support

- **Desktop**: Chrome, Edge (full support), Safari macOS 17+ (Add to Dock)
- **Mobile**: Chrome, Edge, Opera, Samsung Internet (Android), Safari (iOS 16.4+)
- **Note**: Firefox doesn't support PWA installation with manifest
